# i18n Design — Murmur

Date: 2026-03-04
Branch: feature/i18n
Languages: zh-Hans (default) + en

## Goal

Make all hardcoded Chinese UI strings localizable so the app displays in English on English-locale systems.

## Approach

Standard SPM localized resources with short dot-notation keys.

### 1. Package.swift

- Add `defaultLocalization: "zh-Hans"` at the package level.
- Add `resources: [.process("Resources")]` to the `murmur` executable target.

### 2. String Files

Create two files under `Sources/App/Resources/`:

```
Sources/App/Resources/
  zh-Hans.lproj/Localizable.strings   ← key = Chinese value
  en.lproj/Localizable.strings        ← key = English value
```

### 3. Code Changes

| Context | How |
|---|---|
| SwiftUI `Text("中文")` | `Text("key")` — SwiftUI auto-resolves LocalizedStringKey |
| `Section("中文")` / `Toggle("中文", ...)` / `Picker("中文", ...)` / `TextField("中文", ...)` | Pass string literal with key directly; SwiftUI treats first arg as LocalizedStringKey |
| `.help("中文")` | `.help("key")` — also LocalizedStringKey |
| `.confirmationDialog("中文", ...)` | `.confirmationDialog("key", ...)` |
| NSMenuItem title | `String(localized: "key")` |
| `win.title = "中文"` | `String(localized: "key")` |
| State strings (saveLabel, syncStatus, etc.) | `String(localized: "key")` |
| Format strings (e.g. "%lld 个热词") | `String(localized: "hotwords.count \(n)")` using string interpolation overload |
| `TimeFilter` enum rawValue (was Chinese) | Change rawValue to English short id; add `var displayName: LocalizedStringKey` |

### 4. Key Naming Convention

`<module>.<scene>.<detail>`

Examples: `menu.history`, `settings.asr.section`, `common.cancel`, `hotwords.count`.

### 5. Files Touched

- `src-swift/Package.swift`
- `src-swift/Sources/App/AppDelegate.swift`
- `src-swift/Sources/App/SettingsView.swift`
- `src-swift/Sources/App/HistoryView.swift`
- `src-swift/Sources/App/HotwordsView.swift`
- `src-swift/Sources/App/StatsView.swift`
- `src-swift/Sources/App/Resources/zh-Hans.lproj/Localizable.strings` (new)
- `src-swift/Sources/App/Resources/en.lproj/Localizable.strings` (new)
